✅ COMPLETE BATTERY DIGITAL TWIN SYSTEM

This package includes a production-ready battery management digital twin with:

  🧠 SOC Model (LSTM)
     - Bidirectional LSTM with dropout
     - Predicts battery State of Charge
     - Input: 5 features × 20 timesteps
     - Output: SOC (0-1)

  📊 SOH Model (XGBoost)
     - Gradient boosted decision trees
     - Predicts battery State of Health
     - Tracks capacity degradation over cycles
     - Output: SOH (0-100%)

  ⏱️ Charging Time Predictor (Random Forest)
     - Estimates charging duration
     - Uses capacity, voltage, energy data
     - Output: Charging time (hours)

  ⚡ ΔV Model (Gradient Boosting)
     - Detects voltage anomalies
     - Identifies pack imbalance and aging
     - Output: Voltage residual (V)

  🔧 Digital Twin Core
     - Integrates all models
     - Real-time state prediction
     - Battery status classification
     - Anomaly detection

  🌐 REST API (FastAPI)
     - Cloud deployment ready
     - Batch predictions
     - Health reports
     - Statistics tracking

  🎯 BMS Integration (Lightweight)
     - Edge deployment optimized
     - Real-time inference
     - Minimal memory footprint
     - Hardware interface ready

  📊 Visualization Suite
     - Training history plots
     - Real-time prediction graphs
     - Model performance comparisons
     - Statistical analysis

⚡ WHAT HAPPENS WHEN YOU RUN IT
═══════════════════════════════════════════════════════════════════════════════

When you run: python digital_twin_pipeline.py

Step 1: DATA LOADING (1 min)
  → Looks for Excel file: 923-MM0130089005G00006-BTS85-227-1-3-67.xlsx
  → If not found: Generates 5000 synthetic data points
  → Loads 4 datasets: time series, cycles, steps, tests

Step 2: PREPROCESSING (1 min)
  → Cleans data (removes NaN, outliers)
  → Normalizes features using MinMaxScaler
  → Creates SOC from capacity data
  → Creates SOH from cycle degradation

Step 3: SOC MODEL TRAINING (2-3 min)
  → Builds Bidirectional LSTM network
  → Creates 20-timestep sequences
  → Trains on normalized features
  → Validates with 20% held-out data

Step 4: SOH MODEL TRAINING (1 min)
  → Builds XGBoost regressor
  → Uses cycle-level features
  → Predicts capacity degradation
  → Reports R² and RMSE metrics

Step 5: CHARGING TIME MODEL (1 min)
  → Builds Random Forest regressor
  → Uses step-level charging data
  → Predicts charging duration
  → Handles missing data gracefully

Step 6: ΔV MODEL TRAINING (1 min)
  → Builds Gradient Boosting regressor
  → Uses current, temperature, resistance
  → Detects voltage anomalies
  → Trained for residual prediction

Step 7: DIGITAL TWIN CREATION (Instant)
  → Integrates all 4 models
  → Sets up prediction pipeline
  → Initializes state history

Step 8: REAL-TIME SIMULATION (2 min)
  → Runs 150 sequential predictions
  → Simulates battery state changes
  → Updates status dynamically
  → Prints progress every 20 steps

Step 9: VISUALIZATION (2 min)
  → Generates training plots
  → Creates prediction graphs
  → Shows model comparisons
  → Saves to plots/ directory

Step 10: MODEL EXPORT (1 min)
  → Saves SOC model as TensorFlow H5
  → Saves SOH model as XGBoost JSON
  → Saves other models as pickle files
  → Creates deployment configuration

Step 11: REPORT GENERATION (Instant)
  → Creates comprehensive summary
  → Lists all metrics and statistics
  → Shows deployment files
  → Provides next steps

TOTAL TIME: 15-20 minutes

📞 SUPPORT & TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Issue: "ModuleNotFoundError: No module named 'tensorflow'"
→ Solution: pip install -r requirements.txt

Issue: "Excel file not found"
→ Solution: Synthetic data will be generated automatically
→ Or: Place your Excel file in d:\Digital Twin\

Issue: "Out of memory"
→ Solution: Reduce data size or batch size in code
→ Or: Close other applications

Issue: "Permission denied" (on Linux/Mac)
→ Solution: chmod +x demo.py digital_twin_pipeline.py

Issue: "Port 8000 already in use" (FastAPI)
→ Solution: Kill other process or use: uvicorn fastapi_deployment:app --port 8001
